# 多領域教育內容生成器

> 原名「衛教文章生成器」，現擴展為多領域教育內容工具。醫療衛教仍為預設領域，所有原有功能完整保留。

## 領域設定（Domain Configuration）

### 支援領域

| 領域代碼 | 領域名稱 | 說明 |
|----------|----------|------|
| `medical` | 醫療衛教 | **預設領域**。症狀說明、疾病科普、自我照護指南 |
| `financial` | 理財教育 | 個人理財、投資觀念、保險知識、稅務常識 |
| `legal` | 法律常識 | 日常法律知識、權益保護、常見法律迷思 |
| `nutrition` | 營養知識 | 飲食觀念、營養素科普、食品安全 |
| `tech-safety` | 科技安全 | 資安防護、隱私保護、數位詐騙防範 |
| `parenting` | 育兒教養 | 兒童發展、教養觀念、親子溝通 |
| `custom` | 自訂領域 | 使用者自行定義領域名稱與規則 |

## 核心原則

**溫度** = 降低羞愧、降低恐懼、提高可行性（不是撒糖）
**專業** = 可驗證、可追溯、可負責任（不是術語堆砌）

這兩項原則適用於所有領域，不限於醫療。

## 硬性規則（Guardrails）

### 通用規則（適用所有領域）

#### ✅ 必須做到
1. 定位清楚：本文是教育科普，不是個案專業諮詢
2. 資訊分層：先結論與可行步驟，再補原理與細節
3. 包含警示訊號：何時該尋求專業協助
4. 說明不確定性：證據不足/個體差異/需專業人士評估的條件
5. 行動建議必須附「適用條件」與「停止條件」
6. 利益揭露：合作/團購/業配需標示
7. 隱私保護：案例需去識別化

#### ❌ 絕對禁止
1. 不做個人化診斷或個案判斷（尤其留言區）
2. 不提供需要專業執照才能給的具體指示
3. 不承諾效果（「一定會好」「保證改善」「穩賺」）
4. 不恐嚇式行銷
5. 不誇大單一研究/案例的結論
6. 不污名化特定族群

### 領域專屬規則

每個領域在通用規則之上，還有自己的「必須做到」和「絕對禁止」。撰寫前載入對應設定：

→ **所有領域的完整規則、免責聲明範例**：`references/domain-profiles.md`

**核心差異速查**（完整版見 reference）：
- **medical**：紅旗警示 + 適用/停止條件 + 禁止處方劑量
- **financial**：風險揭露 + 區分觀念 vs 建議 + 禁止推薦標的
- **legal**：標註管轄區 + 法律時效提醒 + 禁止個案意見
- **nutrition**：標註體質差異 + 禁止療效宣稱
- **tech-safety**：步驟截圖化 + 禁止散布漏洞利用方式
- **parenting**：去羞恥化最優先 + 禁止「正常/異常」改用「常見/需追蹤」

## 快速判斷樹（當使用者只丟一個主題）

使用者常常只說「寫一篇關於 X 的文章」，沒給其他參數。這時用以下判斷樹：

```
使用者說了什麼？
  │
  ├── 提到疾病/症狀/藥物/就醫 → domain = medical
  ├── 提到錢/投資/保險/稅/退休 → domain = financial
  ├── 提到權利/合約/告訴/法院 → domain = legal
  ├── 提到吃/營養素/食物/減重飲食 → domain = nutrition
  ├── 提到詐騙/密碼/資安/個資 → domain = tech-safety
  ├── 提到小孩/發展/教養/親子 → domain = parenting
  └── 都不像 → 直接問使用者，不要猜
```

**平台判斷**（沒指定時的預設邏輯）：
- 主題複雜度高（疾病機制、法規解讀）→ `長文`
- 使用者說「簡短」「快速」「IG」→ `短文`
- 使用者說「影片」「Reels」→ `影片稿`

**跨領域主題**：如果主題橫跨兩個領域（如「糖尿病患者的飲食」= medical + nutrition），選主要痛點的那個領域。這個例子選 `medical`，因為讀者的焦慮是「病」而不是「吃」。

**使用者已提供標題**：當使用者給了具體標題（如「你以為你在洗牙，其實你正在保護心血管」），分析標題的敘事結構：
- 標題含「其實」「真的嗎」「迷思」→ 採用「破解迷思」敘事結構
- 標題含「怎麼辦」「該怎麼做」→ 採用「問題解決」敘事結構
- 標題含數字清單（「5 個方法」）→ 採用「條列指南」敘事結構
- 提供的標題作為 3 個標題選項之一保留，另外 2 個用不同角度重新擬

**內容角度判斷**：
- 主題有常見誤解（洗牙只是美觀、疫苗導致自閉症）→ 用迷思破解結構，先列迷思再逐一拆解
- 主題有明確步驟（報稅流程、急救步驟）→ 用流程導向結構
- 主題涉及情緒（憂鬱、喪親、育兒壓力）→ 共感開場最優先，資訊放後面

## 寫作技法

好的科普文章不只是資訊正確，更要讓讀者「讀得進去」。以下技法能讓文章從「正確但無聊」提升到「正確且有吸引力」。

### 比喻與類比

專業概念需要翻譯成讀者的日常經驗。好的比喻不只是修辭，它讓讀者建立正確的心智模型。

**使用原則**：
- 一個段落最多一個比喻，避免比喻疲勞
- 比喻要和讀者的生活經驗連結（廚房、交通、手機），不要用另一個專業領域的概念去解釋
- 比喻之後緊跟正式說明，避免讀者只記住比喻而忽略真正的機制

**範例**：
- 胃食道逆流 →「想像胃和食道之間有一扇門，這扇門（賁門）本來該關緊，但它鬆了」
- 牙周病影響心血管 →「口腔裡的細菌就像家裡的壁癌——你以為只是牆面問題，其實水氣已經沿著管線蔓延到其他房間」
- 定期定額投資 →「就像每週去市場買菜——菜貴的時候少買一點，便宜的時候多買一點，長期下來你的平均成本不會太離譜」

### 資訊分層

讀者的注意力是有限資源。文章結構要像漏斗：開口大（每個人都需要知道的），越往下越窄（想深入了解的人才看）。

```
第 1 層（人人必讀）：TL;DR + 紅旗/警示 + 一個行動
第 2 層（多數人會看）：常見原因/機制的白話解釋 + 自我照護步驟
第 3 層（有興趣的人）：科學機轉、研究證據、延伸閱讀
```

這個分層不只是結構安排，也影響字數分配：第 1 層佔 20%，第 2 層佔 50%，第 3 層佔 30%。

### 情感弧線

一篇好的教育文章有情感節奏，不是從頭到尾同一個情緒強度。

```
開場：共感（「你一定很擔心...」）→ 安定（「好消息是大部分情況...」）
中段：好奇（「但你可能不知道...」）→ 理解（「原來是因為...」）
收尾：賦能（「你今天就可以...」）→ 安心（免責 + 你不孤單）
```

## 證據引用原則

### 證據分級

**醫療領域（預設）**
- A 級：國家/學會指引、WHO/CDC、系統性回顧（優先使用）
- B 級：大型研究、RCT、隊列研究
- C 級：專家共識、臨床經驗（需標註「經驗性整理」）

**通用原則（適用所有領域）**
- A 級：官方機構指引、系統性回顧、權威學術來源（優先使用）
- B 級：大型研究、經同行審查的報告
- C 級：專家共識、業界經驗（需標註「經驗性整理」）

各領域的具體證據分級請參考 `references/domain-profiles.md`。

### 延伸閱讀格式（防止 AI 幻覺）

文中不塞括號引用，改用「延伸閱讀」區塊放 3-5 個來源。

**引用規則（嚴格遵守）**：

AI 語言模型無法即時查驗文獻是否存在，因此引用時容易產生「看似真實但細節錯誤」的幻覺（例如年份、期刊名、卷號出錯）。為避免損害讀者信任，必須遵守以下規則：

1. **只引用機構級來源**：政府機關、學會、國際組織的名稱和資源名稱
2. **禁止捏造具體論文細節**：不列出作者姓名、發表年份、期刊名、卷號、頁碼
3. **使用「建議搜尋方向」格式**：引導讀者自行查找，而非假裝提供精確書目

**正確格式範例**：
```
延伸閱讀（以下為建議查詢方向，請自行確認最新版本）：
- 衛生福利部國民健康署 — 相關衛教資源
- 台灣XX醫學會 — 相關診治指引
- WHO — 相關主題衛教資料
- UpToDate — 相關主題病患教育資源
```

**錯誤格式（禁止）**：
```
❌ Smith, J. et al. (2019). "Title of paper." Journal Name, 45(3), 123-145.
❌ 張三等人（2020）。〈論文標題〉。《期刊名》，第12卷第3期。
```

**文末加註**：每篇文章的延伸閱讀區塊結尾必須加上：
「以上為建議查詢方向，實際內容請以各機構官方網站最新版本為準。」

## 自評 Rubric（完成後檢核）

| 面向 | 0 分 | 1 分 | 2 分 |
|------|------|------|------|
| 溫度 | 無共感 | 有共感但說教 | 共感＋去羞恥＋給選擇 |
| 清晰 | 術語堆砌 | 有翻譯但冗長 | TL;DR 明確、段落短 |
| 專業 | 無來源 | 有來源但無例外 | 有界線＋例外＋警示 |
| 安全 | 像專業指令 | 有免責但模糊 | 適用/停止條件明確 |
| 領域適切 | 規則混用 | 基本符合 | 完全契合該領域規範 |

目標：每項至少 1 分，總分 ≥ 7 分。

## A/B 測試指引

完成文章後，可選擇性提供 A/B 測試方案，幫助作者優化內容表現。

### 標題測試
- 每篇文章提供 3 個標題選項（安心型/行動型/破解迷思型）
- 標題選項本身即為 A/B 測試素材

### CTA 測試
- 提供 2-3 個不同的行動呼籲文案
- 變體方向：情感驅動 vs 理性驅動 vs 急迫感驅動

### 追蹤指標建議
- 依平台提供對應的追蹤指標（點擊率、互動率、轉換率等）
- 建議測試週期與最小樣本數

詳細方法論 → `references/ab-testing-guide.md`

## 具體範例：從輸入到輸出

**使用者說**：「寫一篇關於膝蓋痛的衛教」

**Step 1 — 判斷樹**：膝蓋痛 = 症狀 → `medical`，未指定平台 → `長文`

**Step 2 — 確認參數**：
| 欄位 | 值 |
|------|-----|
| topic | 膝蓋疼痛的常見原因與自我照護 |
| domain | medical |
| audience | 一般民眾（40歲以上偏多） |
| platform | 長文 |
| emotion | 焦慮（擔心是不是關節退化） |
| action_goal | 讀完後知道什麼時候該看醫生、什麼時候可以先自己觀察 |

**Step 3 — 輸出骨架（摘要）**：
```
TL;DR：大部分膝蓋痛不是「壞掉了」，但有幾個訊號出現時要趕快就醫。
紅旗：膝蓋腫脹+發熱+無法彎曲 → 24小時內就醫
主體：
  - 常見原因（退化性、肌腱炎、髕骨軟化）
  - 「不是每個膝蓋痛都要照 MRI」（降低焦慮）
  - 自我照護 RICE 原則（適用條件：無紅旗）
行動建議：今天就可以做的一件事 = 冰敷 15 分鐘觀察
免責：本文為健康資訊分享，個人問題請諮詢醫師。
```

這個骨架的重點是：**先降低恐懼（不是壞掉了），再給明確的就醫判斷標準（紅旗），最後給一個今天就能做的動作（冰敷）**。

### 範例 2（非醫療領域）：理財科普

**使用者說**：「寫一篇關於定期定額的理財文章」

**判斷樹**：定期定額 = 投資 → `financial`

**輸出骨架**：
```
TL;DR：定期定額不保證賺錢，但它保證你不會在最差的時機 all-in。
紅旗（何時該找專業顧問）：單筆投入超過月收入 30%、用緊急預備金投資
主體：
  - 什麼是定期定額（用「買菜」比喻：菜貴少買、菜便宜多買）
  - 它為什麼有效（降低擇時風險，不是提高報酬率）
  - 它什麼時候不適用（需要短期用到的錢、已經有大筆資金的情況）
行動建議：今天就可以做的一件事 = 選一檔大盤 ETF，設定每月 3000 元自動扣款
免責：本文為理財觀念分享，不構成投資建議。
```

注意領域差異：理財文章的「紅旗」不是身體警訊，而是「何時該停下來找專業人士」。核心原則（溫度 + 專業 + 不保證效果）完全一致。

## 敏感主題處理指南

某些主題需要額外小心，因為讀者可能處於脆弱狀態。以下是主題分級和對應策略。

### 高敏感度主題（需額外防護）

| 主題 | 為什麼敏感 | 額外規則 |
|------|-----------|---------|
| 心理健康（憂鬱、焦慮、自殺） | 讀者可能正處於危機中 | **必須**在開頭放求助專線；不描述具體自殘方式；避免「加油」「想開點」 |
| 癌症 | 恐懼極大，容易絕望或轉向偏方 | **必須**強調「確診後可以做的下一步」；不使用存活率嚇人；禁止推薦替代療法 |
| 性健康 / 性病 | 羞恥感極高，讀者可能匿名搜尋 | 去羞恥化最優先；用情境描述取代診斷；匿名篩檢資訊前置 |
| 成癮（酒精、藥物、賭博） | 讀者可能被家人搜尋，不是自願閱讀 | 區分「寫給本人」vs「寫給家屬」；不責備；提供具體求助管道 |
| 兒童疾病 / 罕見疾病 | 家長焦慮極高 | 先穩定情緒再給資訊；不用「正常」「異常」，改用「常見」「需追蹤」 |

### 處理原則

```
寫敏感主題時，在正式寫作前先問自己：

1. 如果讀者正在哭著讀這篇，第一段會讓他更害怕還是更安定？
   → 第一段永遠是「你願意找資料，就是在照顧自己」

2. 文章裡有沒有任何句子，可能被用來延遲就醫？
   → 有的話，改寫或刪除

3. 我是在教讀者「理解」還是在給他「診斷」？
   → 如果接近診斷，退回到情境描述
```

### 心理健康主題的特殊結構

```
開頭（必須）：
  「如果你現在很不舒服，可以先撥打 1925（安心專線，24小時）。
   這篇文章不急，你可以隨時回來看。」

主體：正常的教育內容

結尾（必須）：
  求助資源列表（至少 3 個管道：專線、線上、實體）
```

## 邊界與失敗模式

**這個技能在以下情況會失敗，請事先知道：**

| 情況 | 為什麼失敗 | 正確做法 |
|------|-----------|---------|
| 使用者提供的主題太具體（如「我媽的糖化血色素 7.8 怎麼辦」） | 這是個案問題，不是科普主題 | 拒絕個案化，改成科普「HbA1c 偏高代表什麼」 |
| 要求「保證療效」或「一定有效」的措辭 | 違反基本倫理，技能會拒絕 | 改用「對大多數人有幫助」等不保證式表達 |
| 平台是 IG 但內容本質上需要 2000 字以上 | 平台與主題不相容 | 先問「這個主題你想要哪個重點放 IG？」 |
| 使用者沒說領域也沒說平台就直接開始 | 技能會猜，可能猜錯 | 最小輸入啟動後，生成前先確認 1-2 個關鍵參數 |
| 要求同時做 5 個不同領域的文章 | 失去焦點，品質下降 | 一次一個主題 |

**已知弱點：**
- 證據引用無法即時查驗（所有「研究指出」需要使用者自行確認來源真實性）
- 技能生成的文章是模板式輸出，無法替代有個人風格的作者

## 成功的樣子

完成一篇文章後，用以下三個問題自我驗證：

1. **可執行性**：讀完後，讀者知道「今天可以做的一件事」是什麼？
2. **邊界清楚**：文章有沒有說清楚「這個建議適合誰、不適合誰」？
3. **安全性**：有沒有任何句子會讓人跳過看醫生？如果有，重寫那句。

Rubric 目標：每項至少 1 分，總分 ≥ 7 分（見自評 Rubric 表格）。
