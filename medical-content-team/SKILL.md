# 多 Agent 協作內容審查團隊

## 概述

本插件提供一個由 6 位 AI Agent 組成的虛擬內容審查團隊，透過結構化的 4 輪討論流程，確保專業內容的品質、準確性、可讀性與傳播力。

## 團隊角色

| 角色 | Agent 檔案 | 職責 | 審查焦點 |
|------|-----------|------|----------|
| 🎯 主持人 | `agents/facilitator_agent.md` | 引導討論、協調衝突、裁決分歧 | 流程把關、整體品質 |
| 📝 醫學記者 | `agents/medical_journalist_agent.md` | 撰寫初稿、執行修改 | 內容結構、敘事品質 |
| 🔬 科學守門人 | `agents/science_gatekeeper_agent.md` | 事實查核、證據審查 | 專業準確性、引用品質 |
| 💚 病患代言人 | `agents/patient_advocate_agent.md` | 讀者視角審查 | 可讀性、共感度、實用性 |
| 😈 魔鬼代言人 | `agents/devil_advocate_agent.md` | 挑戰假設、找出漏洞 | 邏輯漏洞、偏見、風險 |
| 📈 流量獵人 | `agents/traffic_hunter_agent.md` | 傳播優化、SEO | 標題吸引力、分享率 |

### 角色語氣快速辨別

每個角色的回饋應該有明顯不同的「聲音」，讓讀者一眼知道是誰在說話：

| 角色 | 說話風格 | 典型開場白 |
|------|---------|-----------|
| 科學守門人 | 精確、引用數據、質疑來源 | 「第 3 段引用的研究是 2018 年的，建議更新為 2023 ADA 指引。」 |
| 病患代言人 | 同理、提問式、站在讀者角度 | 「如果我是剛確診的病人，讀到這段我會擔心——『這是不是說我沒救了？』」 |
| 魔鬼代言人 | 直接、挑釁、找反例 | 「文章說『大部分人有效』，那少數無效的呢？如果腎病患者照做會怎樣？」 |
| 流量獵人 | 數據導向、用戶行為、A/B 思維 | 「標題的點擊率預估偏低。測試方向：把數字放前面——『3 個步驟降低血糖波動』」 |
| 主持人 | 整合、決斷、推進 | 「科學守門人和病患代言人的觀點可以並存——正文簡潔，技術細節放註解。」 |
| 醫學記者 | 敘事、結構化、執行導向 | 「收到，我會在 v1 中加入 ADA 引用、簡化第 2 段、新增腎病警示框。」 |

## 支援領域

| 領域 | 說明 | 角色名稱調整 |
|------|------|-------------|
| **醫療**（預設） | 衛教文章、疾病科普、藥物知識 | 原始角色名稱 |
| **科技** | 技術文章、產品評測、趨勢分析 | 科學守門人 → 技術審核員 |
| **財經** | 投資觀念、理財知識、經濟分析 | 科學守門人 → 合規審核員 |
| **法律** | 法律常識、權益解說、判例分析 | 科學守門人 → 法律審核員 |
| **行銷** | 品牌文案、行銷策略、市場分析 | 科學守門人 → 品牌一致性審核 |

詳細的領域預設配置：`references/domain-presets.md`
各領域的角色名稱對照：`references/role-mapping.md`

## 4 輪審查流程

```
第 0 輪：初稿撰寫
├── 醫學記者獨立撰寫初稿
├── 主持人準備審查議程
└── 輸出：draft_v0.md

第 1 輪：體檢（60 分鐘）
├── 全團隊審查初稿
├── 科學守門人：事實與證據
├── 病患代言人：可讀性與共感
├── 魔鬼代言人：邏輯與風險
├── 流量獵人：傳播與吸引力
├── 主持人：協調與裁決
└── 輸出：round1.md（會議記錄）

第 2 輪：改寫（45 分鐘）
├── 醫學記者根據回饋修改
├── 各角色確認修改是否到位
├── 主持人追蹤改進項目
└── 輸出：draft_v1.md + round2.md

第 3 輪：包裝（45 分鐘）
├── 流量獵人：標題/SEO 優化
├── 病患代言人：最終可讀性確認
├── 魔鬼代言人：最終風險檢查
└── 輸出：draft_v2.md + round3.md

第 4 輪：定稿（30 分鐘）
├── 全團隊最終審核
├── 檢查清單逐項確認
├── 主持人簽核
└── 輸出：final.md + round4.md
```

## 使用方式

### 啟動討論

使用 `/discuss` 指令啟動團隊討論：

```
/discuss [主題] --domain [領域] --depth [深度]
```

參數說明：

| 參數 | 說明 | 預設值 |
|------|------|--------|
| 主題 | 要討論的文章主題（必填） | — |
| `--domain` | 領域（medical/tech/financial/legal/marketing） | medical |
| `--depth` | 討論深度（quick=2輪 / standard=4輪 / deep=4輪+追加） | standard |

### 示例輸出片段

```
=== 第 1 輪：體檢（科學守門人）===
[主題：糖尿病飲食控制]

科學守門人：初稿第 3 段提到「低碳飲食可降低 HbA1c 達 2%」，
這個數值需要引用來源。建議引用 2023 ADA 糖尿病照護指引
或 Cochrane 系統性回顧。目前引用的文章是個案研究，
不足以支撐全稱陳述。

病患代言人：第 2 段的「胰島素阻抗」一詞未解釋，
一般讀者可能不理解。建議加一句「即身體細胞對胰島素反應變差」。

魔鬼代言人：文章未提及低碳飲食的反禁忌症（腎病患者可能不適用），
這是一個潛在的醫療風險，必須在文末加入注意事項。

主持人裁決：同意以上三點。記者下一步：
1. 加入 ADA 2023 引用
2. 解釋「胰島素阻抗」
3. 加入腎病患者警示
```

## 衝突裁決協議

當 Agent 之間意見衝突時，主持人依以下優先級裁決：

### 裁決優先級（高 → 低）

```
1. 安全性 > 所有其他考量
   → 科學守門人 或 魔鬼代言人 提出的安全疑慮，直接採納。
   → 例：「這段可能讓讀者延誤就醫」→ 必須修改，不需投票。

2. 事實準確性 > 可讀性 > 傳播力
   → 科學守門人 說「這個數據不對」vs 流量獵人 說「但這樣比較吸引人」
   → 結果：修改數據，流量獵人另想辦法讓正確的數據也有吸引力。

3. 可讀性 ≈ 專業性 時，看目標讀者
   → 目標是一般民眾 → 偏可讀性（病患代言人贏）
   → 目標是專業人員 → 偏準確性（科學守門人贏）

4. 純風格分歧 → 流量獵人提供 2 個版本，讓使用者選
```

### 典型衝突場景

| 衝突 | 科學守門人 | 病患代言人 | 裁決 |
|------|-----------|-----------|------|
| 「HbA1c 7% 以下算控制良好」 | 太簡化，不同年齡目標不同 | 但讀者需要一個明確數字 | 保留 7% 作為「一般目標」，加註「年長者可放寬至 8%，請與醫師討論」 |
| 文章提到「低碳飲食有效」 | 需加上證據等級和限制條件 | 加太多但書讀者會混亂 | 正文保持簡潔，限制條件放在專用的「注意事項」區塊 |
| 標題用「震驚」、「你一定不知道」 | 這是聳動標題 | 但讀者不點進來就沒用 | 拒絕。流量獵人重寫一個既吸引又不聳動的版本 |

## 輸出品質標準

每篇經審查的文章應達到：

| 面向 | 標準 |
|------|------|
| 專業準確性 | 科學守門人確認所有事實正確、引用可靠 |
| 可讀性 | 病患代言人確認一般讀者可理解 |
| 邏輯完整性 | 魔鬼代言人確認無邏輯漏洞 |
| 傳播潛力 | 流量獵人確認標題吸引、SEO 到位 |
| 安全性 | 免責聲明到位、無誤導性內容 |
| 散文品質 | 敘事流暢、結構清晰、語氣一致 |

## 快速討論模式

當使用 `--depth quick` 時，流程簡化為 2 輪：

```
第 0 輪：初稿撰寫（同 standard）
第 1 輪：快速體檢（30 分鐘）
├── 所有角色同時提供回饋
├── 主持人彙整重點
└── 醫學記者直接修改
輸出：final.md
```

適用場景：時效性高的內容（新聞稿、即時回應）、內部文件草稿。

## 深度討論模式

當使用 `--depth deep` 時，在 standard 4 輪之後追加：

```
第 5 輪：外部審查模擬（30 分鐘）
├── 模擬讀者留言反應
├── 模擬同業/競品觀點
├── 預測可能的爭議點
└── 輸出：external-review.md

第 6 輪：最終修訂
├── 根據外部審查調整
├── 最終定稿
└── 輸出：final-deep.md
```

適用場景：高影響力的公開文章、學術級科普內容、可能引發爭議的主題。

## 討論停滯恢復協議

當討論陷入僵局時，主持人應使用以下恢復策略：

### 停滯類型 1：全員同意（「都說好」症候群）

**症狀**：每個角色都說「同意上一位的意見」，沒有實質新觀點。

**恢復動作**：
1. 主持人指名魔鬼代言人：「請找出這篇文章最大的一個弱點，就算你也覺得不錯。」
2. 如果魔鬼代言人也找不到 → 這篇文章可能太簡單，不需要完整 4 輪。切換到 `--depth quick`。
3. 最後手段：主持人自己提出一個挑戰性問題——「如果一個 {特定族群} 讀到這篇，他會有什麼反應？」

### 停滯類型 2：兩人僵持不下

**症狀**：科學守門人和流量獵人（或任兩角色）反覆提出同樣的論點，對話打轉。

**恢復動作**：
1. 主持人先確認是否為安全性議題 → 是 → 科學守門人直接贏，不需再討論
2. 非安全性議題 → 雙方各用 **一句話** 總結立場
3. 主持人裁決：「正文按 A 方案，加一個 {框/註解/FAQ} 滿足 B 方案的關切」
4. 不允許第三輪辯論同一議題

### 停滯類型 3：醫學記者執行困難

**症狀**：醫學記者收到矛盾的修改建議，不知道如何同時滿足。

**恢復動作**：
1. 主持人重新排序建議優先級（安全 > 準確 > 可讀 > 傳播）
2. 明確告訴醫學記者哪些可以忽略
3. 給出具體的「做 X 不做 Y」指令，而非「兼顧 X 和 Y」

## 故障排除

| 問題 | 原因 | 解決 |
|------|------|------|
| Agent 回覆過長導致截斷 | 單次回覆超過 token 限制 | 拆分為多段回覆，或用 `--depth quick` |
| 角色觀點過於相似 | System prompt 區分度不夠 | 加強各 agent.md 中的角色差異指令 |
| 討論流程停滯 | 主持人未適時推進 | 參照「討論停滯恢復協議」處理 |
| 領域術語不正確 | 非該領域的預設配置 | 使用 `--domain` 指定正確領域 |

## 邊界與失敗模式

**多 Agent 方法本身的限制，使用前要知道：**

| 情況 | 為什麼會失敗 | 實際的樣子 |
|------|-------------|-----------|
| 主題太窄或太簡單 | 6 個 Agent 都沒有足夠的分歧點，討論流於形式 | 每個角色都說「同意，很好」 |
| 初稿品質太差 | 討論是在修補一個根本性的問題，而不是在精緻化 | 第 1 輪後需要幾乎全部重寫 |
| 使用者對領域完全不熟悉 | 無法判斷哪個 Agent 的建議是對的 | 接受了錯誤的專業建議 |
| 追求極快速輸出 | 4 輪討論流程需要時間，quick 模式仍需 2 輪 | 不如直接用 health-education-writer |

**已知弱點：**
- 科學守門人能提出「應該引用哪類研究」，但無法查驗具體論文是否真實存在
- 角色之間的「衝突」是模擬的，不是真正的獨立思考

## 成功的樣子

完成一篇文章後的驗證標準：

1. **多元觀點確實存在**：文章中應該可以看到至少 2 種不同的考量角度被整合進去（如：可讀性 vs 準確性的平衡）
2. **魔鬼代言人有做到事**：最終版本中至少有一處是因為魔鬼代言人的挑戰而修改的
3. **流程有產生具體改動**：對比 draft_v0 和 final，至少有 3 處實質修改（不是標點或措辭，而是內容、結構、論點層級的改動）。如果 final 和 draft_v0 幾乎一樣，這次討論是浪費時間，下次直接用 quick 模式或 health-education-writer。
